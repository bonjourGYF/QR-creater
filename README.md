# 挪车联系二维码网站

这是一个纯静态单页网站，用来生成“网页链接二维码”。微信扫码后打开网页联系卡片，避免微信无法展示纯文本二维码的问题。

## 功能

- 首页填写标题、车主称呼、电话号码、微信号、车牌号、提示语。
- 点击或实时输入后生成二维码，二维码内容是带 `mode=card` 和联系信息 query 参数的 URL。
- `mode=card` 页面只展示适合手机查看的挪车联系卡片。
- 支持一键拨打电话、复制微信号、复制生成链接、下载 512 x 512 PNG、打印挪车卡片。
- 不需要后端、不需要数据库、不依赖外部资源。
- 所有用户输入展示前会进行 HTML 转义，降低 XSS 风险。

## 本地测试

1. 直接双击打开 `index.html` 可以预览页面和生成二维码。
2. 如果是 `file:///` 本地打开，页面会提示：本地文件链接不能直接给别人扫码使用，必须部署到线上后再生成最终二维码。
3. 运行自动检查：

```bash
node --test tests/static-site.test.js
```

4. 也可以启动一个本地静态服务测试 URL 模式：

```bash
python -m http.server 8080
```

然后访问 `http://localhost:8080/`，填写联系方式并生成二维码。

## GitHub Pages 部署

1. 新建一个 GitHub 仓库。
2. 上传 `index.html`、`README.md` 和 `tests/`。
3. 打开仓库的 `Settings`。
4. 进入 `Pages`。
5. 在 `Build and deployment` 中选择 `Deploy from a branch`。
6. 分支选择 `main`，目录选择 `/root`，保存。
7. 等待 GitHub Pages 给出线上地址，例如：

```text
https://你的用户名.github.io/仓库名/
```

8. 用线上地址打开页面，再生成最终二维码。此时二维码中的链接会使用当前 `http://` 或 `https://` 地址作为基础链接。

## 使用建议

建议只填写电话或微信，不要填写住址、身份证等敏感信息。打印前可以先用手机扫码确认能打开联系卡片，并检查一键拨打电话和复制微信号是否符合预期。
